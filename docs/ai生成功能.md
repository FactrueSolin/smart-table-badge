# AI 生成功能

## 概述

HTMLPush 内置 AI HTML 生成功能，支持通过自然语言描述自动生成/编辑符合手机横屏展示规范的 HTML 页面。

## 使用方式

1. 进入管理后台 → 代码编辑 标签页
2. 在 AI 输入栏中用自然语言描述你想要的页面效果
3. 点击 "AI 生成" 按钮，或按 Enter 键
4. AI 会流式输出 HTML 代码，实时显示在左侧编辑器和右侧预览区
5. 如需中断生成，点击 "停止" 按钮
6. 满意后点击 "保存" 将页面保存到服务器

## 工作原理

- **上下文感知**：AI 会读取当前编辑器中的 HTML 代码作为上下文，基于现有代码进行修改；如果编辑器为空，则从零生成
- **规范约束**：系统自动将 [`手机展示HTML规范.md`](手机展示HTML规范.md) 的内容作为 system prompt 发送给 AI，确保生成的 HTML 符合手机横屏全屏展示的要求
- **流式输出**：AI 逐 token 返回内容，前端实时更新编辑器和 iframe 预览，无需等待生成完成
- **可打断**：使用 AbortController 实现随时中断生成

## 环境变量配置

在 `.env` 文件中配置 AI 服务连接信息：

```env
# AI 配置（OpenAI 兼容 API）
AI_BASE_URL=https://api.openai.com/v1
AI_MODEL=gpt-4o
AI_API_KEY=sk-xxx
```

| 变量 | 说明 | 示例 |
|------|------|------|
| `AI_BASE_URL` | AI 服务的 API 地址（OpenAI 兼容格式） | `https://api.openai.com/v1` |
| `AI_MODEL` | 使用的模型名称 | `gpt-4o`, `claude-3-5-sonnet` |
| `AI_API_KEY` | API 密钥 | `sk-xxx` |

## 支持的 AI 服务

任何兼容 OpenAI Chat Completions API 格式的服务均可使用，包括但不限于：

- OpenAI API
- Azure OpenAI
- 各类第三方代理（需确保支持 `/chat/completions` 端点和 SSE 流式输出）

## 提示词技巧

- **描述布局**：描述页面的整体结构，如"三列统计卡片"、"左侧图片右侧文字"
- **指定颜色**：说明背景色、文字色等视觉元素
- **说明用途**：告诉 AI 页面的用途，如"用于展示实时数据的仪表盘"
- **迭代修改**：先生成基础版本，再逐步描述修改需求，如"把标题字体放大"、"增加一个底部导航栏"

## 注意事项

- AI 生成的代码可能需要人工审查和调整
- 确保 AI_API_KEY 的安全性，不要提交到版本控制系统
- 流式生成过程中请勿关闭页面，否则可能导致生成中断
