# HTMLPush 项目架构

## 项目目标

做一个 Web 服务，让手机通过浏览器变成一个"桌面显示器"——后台上传/切换 HTML 内容，手机端实时自动更新显示。

核心场景：在管理后台上传 HTML 文件 → 选择当前展示的文件 → 手机浏览器首页自动刷新为最新内容。

## 技术选型

| 层面 | 选择 | 说明 |
|------|------|------|
| 框架 | Next.js 16 (App Router) | 前后端一体，Route Handlers 做 API |
| 前端 | React 19 + Tailwind CSS 4 | 管理页面用 React 组件 |
| 实时通信 | SSE (Server-Sent Events) | 后台切换内容时，手机端自动更新 |
| 数据存储 | 本地文件系统 | HTML 文件存 `data/pages/`，元数据存 `data/config.json` |
| API 规范 | OpenAPI | RESTful 风格接口 |

## API 设计

### 页面管理

| 方法 | 路径 | 说明 |
|------|------|------|
| `GET` | `/api/pages` | 获取所有已上传的HTML页面列表 |
| `POST` | `/api/pages` | 上传新的HTML文件（multipart/form-data） |
| `GET` | `/api/pages/[id]` | 获取指定页面的HTML内容 |
| `DELETE` | `/api/pages/[id]` | 删除指定页面 |

### 展示控制

| 方法 | 路径 | 说明 |
|------|------|------|
| `GET` | `/api/current` | 获取当前展示的页面信息 |
| `PUT` | `/api/current` | 切换当前展示的页面（body: `{ pageId: string }`） |

### 实时推送

| 方法 | 路径 | 说明 |
|------|------|------|
| `GET` | `/api/sse` | SSE 连接端点，推送内容变更事件 |

SSE 事件格式：
```
event: content-changed
data: {"pageId": "xxx", "timestamp": 1234567890}
```

## 核心流程

### 实时推送机制

采用 SSE 而非 WebSocket，原因：单向推送场景更简单、HTTP 原生支持、自动重连。

```mermaid
sequenceDiagram
    participant Phone as 手机浏览器
    participant SSE as SSE端点
    participant Admin as 管理页面
    participant API as API Handler
    participant Store as 本地存储

    Phone->>SSE: 建立SSE连接
    Admin->>API: PUT /api/current 切换页面
    API->>Store: 更新 config.json
    API->>SSE: 触发 content-changed 事件
    SSE->>Phone: 推送 content-changed
    Phone->>API: GET /api/current 获取新内容
    Phone->>Phone: 用 iframe 或 innerHTML 渲染新HTML
```

### 首页展示方案

首页使用 `iframe` 嵌入当前 HTML 内容，全屏显示。收到 SSE 事件后自动刷新 iframe 的 src。这样做的好处是：上传的 HTML 完全隔离渲染，不会影响主页面的 SSE 连接和逻辑。

首页额外功能：
- **全屏模式**：点击"全屏"按钮进入浏览器全屏，并尝试锁定横屏方向
- **横屏提示**：竖屏时显示"请横屏放置手机"的提示遮罩
- **自动隐藏控制栏**：全屏模式下 3 秒无操作自动隐藏顶部控制栏
- **管理后台入口**：顶部左侧提供"管理后台"按钮，快速跳转 `/admin`

### SSE 事件管理器

在 [`lib/sse.ts`](lib/sse.ts) 中实现一个简单的发布/订阅模式：
- 维护一个 `Set<ReadableStreamDefaultController>` 存储所有活跃的 SSE 连接
- API 切换内容时调用 `broadcast()` 向所有连接推送事件
- 连接断开时自动从 Set 中移除

## 管理页面功能

管理页面 `/admin` 提供以下操作：
- 查看已上传的 HTML 页面列表
- 上传新的 HTML 文件（拖拽或选择文件）
- 预览某个 HTML 页面
- 切换当前展示的页面（一键切换）
- 删除不需要的页面

## 管理页面功能

管理页面 `/admin` 提供以下操作：
- **密码认证**：通过 `ADMIN_PASSWORD` 环境变量设置密码，cookie 认证有效期 7 天
- 查看已上传的 HTML 页面列表
- 上传新的 HTML 文件（拖拽或选择文件）
- **代码编辑器**：直接在页面内编写 HTML 代码并保存
- 预览某个 HTML 页面
- 切换当前展示的页面（一键切换）
- 删除不需要的页面

## 手机展示 HTML 规范

详见 [`docs/手机展示HTML规范.md`](手机展示HTML规范.md)。核心要点：
- 使用 `viewport` meta 标签禁止缩放
- `html, body` 设置 `width: 100%; height: 100%; overflow: hidden`
- 使用 `vw`/`vh` 单位让字体自适应屏幕
- 推荐深色背景 + 高对比度文字

## AI 生成功能

管理后台内置 AI HTML 生成功能，支持自然语言描述自动生成符合手机横屏展示规范的 HTML 页面。

### 工作原理

1. **上下文感知**：AI 读取当前编辑器中的 HTML 代码作为上下文，基于现有代码进行修改；编辑器为空时从零生成
2. **规范约束**：系统自动将 [`手机展示HTML规范.md`](手机展示HTML规范.md) 的完整内容作为 system prompt 发送给 AI
3. **流式输出**：AI 逐 token 返回内容，前端实时更新编辑器和 iframe 预览
4. **可打断**：使用 AbortController 实现随时中断生成

### API 端点

| 方法 | 路径 | 说明 |
|------|------|------|
| `POST` | `/api/ai/generate` | AI 流式生成 HTML（body: `{ prompt: string, currentHtml?: string }`） |

请求体：
```json
{
  "prompt": "做一个三列统计卡片的页面",
  "currentHtml": "<!DOCTYPE html>..."  // 可选，当前编辑器中的代码
}
```

响应：`text/event-stream` 格式，纯文本 HTML 片段，前端直接追加到编辑器。
